Kita sekarang dapat beralih ke instruksi `make`, yang berada di `make.rs` dan akan membuat beberapa fitur berikut:

- Menginisialisasi catatan Escrow dan menyimpan semua persyaratannya.
- Membuat Vault (_Associated Token Account_ untuk `mint_a` yang dimiliki oleh `escrow`).
- Memindahkan Token A milik _maker_ ke _vault_ tersebut dengan melakukan CPI ke program SPL-Token.

### Akun

Akun-akun yang dibutuhkan dalam konteks ini adalah:

- `maker`: pengguna yang menentukan persyaratan dan menyetor token dalam alamat `mint_a` ke dalam `Escrow`
- `escrow`: akun yang menyimpan persyaratan pertukaran (alamat _maker_, alamat _mint_ token, jumlah token)
- `mint_a`: token yang disetor oleh `maker` dalam alamat `mint_a`
- `mint_b`: token yang diinginkan `maker` sebagai pertukaran dalam alamat `mint_b`
- `maker_ata_a`: akun _Associated Token Account_ token yang terkait dengan `maker` dan `mint_a` yang digunakan untuk menyetor token ke dalam `vault`
- `vault`: akun _Associated Token Account_ token yang terkait dengan `escrow` dan `mint_a` yang merupakan tempat penyimpanan token yang disetor oleh _maker_.
- `associated_token_program`: _Associated Token Program_ yang digunakan untuk membuat _Associated Token Account_
- `token_program`: program token yang digunakan untuk fungsi CPI transfer
- `system_program`: program sistem dalam Solana yang digunakan untuk membuat `Escrow`

Implementasi Akun dan semua _constraints_ akan terlihat seperti ini:

```rust
#[derive(Accounts)]
#[instruction(seed: u64)]
pub struct Make<'info> {
    #[account(mut)]
    pub maker: Signer<'info>,
    #[account(
        init,
        payer = maker,
        space = Escrow::INIT_SPACE + Escrow::DISCRIMINATOR.len(),
        seeds = [b"escrow", maker.key().as_ref(), seed.to_le_bytes().as_ref()],
        bump,
    )]
    pub escrow: Account<'info, Escrow>,

    /// Token Accounts
    #[account(
        mint::token_program = token_program
    )]
    pub mint_a: InterfaceAccount<'info, Mint>,
    #[account(
        mint::token_program = token_program
    )]
    pub mint_b: InterfaceAccount<'info, Mint>,
    #[account(
        mut,
        associated_token::mint = mint_a,
        associated_token::authority = maker,
        associated_token::token_program = token_program
    )]
    pub maker_ata_a: InterfaceAccount<'info, TokenAccount>,
    #[account(
        init,
        payer = maker,
        associated_token::mint = mint_a,
        associated_token::authority = escrow,
        associated_token::token_program = token_program
    )]
    pub vault: InterfaceAccount<'info, TokenAccount>,

    /// Programs
    pub associated_token_program: Program<'info, AssociatedToken>,
    pub token_program: Interface<'info, TokenInterface>,
    pub system_program: Program<'info, System>,
}
```

**Catatan**: Instruksi ini hanya menggunakan satu macam `token_program`. Karena fungsi `take` melakukan transfer untuk kedua akun mint, kita harus memastikan kedua akun mint dimiliki oleh `token_program` yang sama (SPL Token atau Token-2022). Jika tidak, maka fungsi CPI akan gagal.

### Logika Program

Setelah proses inisialisasi Akun, kita dapat melakukan implementasi logika dengan membuat fungsi pembantu yang lebih kecil sebagai implementasi dari `struct` akun.

Kita mulai dengan mengisi akun `Escrow` menggunakan fungsi pembantu `set_inner()`, dan kemudian kita akan melanjutkan dengan menyetor token melalui CPI `transfer` seperti kodingan di bawah ini:

```rust
impl<'info> Make<'info> {
    /// # Pembuatan akun Escrow
    fn populate_escrow(&mut self, seed: u64, amount: u64, bump: u8) -> Result<()> {
        self.escrow.set_inner(Escrow {
            seed,
            maker: self.maker.key(),
            mint_a: self.mint_a.key(),
            mint_b: self.mint_b.key(),
            receive: amount,
            bump,
        });

        Ok(())
    }

    /// # Penyetoran token
    fn deposit_tokens(&self, amount: u64) -> Result<()> {
        transfer_checked(
            CpiContext::new(
                self.token_program.to_account_info(),
                TransferChecked {
                    from: self.maker_ata_a.to_account_info(),
                    mint: self.mint_a.to_account_info(),
                    to: self.vault.to_account_info(),
                    authority: self.maker.to_account_info(),
                },
            ),
            amount,
            self.mint_a.decimals,
        )?;

        Ok(())
    }
}
```

Kita dapat melihat bahwa Anchor telah membantu kita dalam beberapa cara:

- `set_inner()`: menjamin setiap bagian data dalam akun `Escrow` terisi.
- `transfer_checked`: membungkus Token CPI sama seperti _System Helper_ yang telah kita gunakan sebelumnya.

Dan sekarang kita dapat melanjutkan dengan membuat fungsi `handler` di mana kita melakukan beberapa verifikasi sebelum menggunakan fungsi pembantu inti seperti ini:

```rust
pub fn handler(ctx: Context<Make>, seed: u64, receive: u64, amount: u64) -> Result<()> {
    // Verifikasi jumlah
    require_gt!(receive, 0, EscrowError::InvalidAmount);
    require_gt!(amount, 0, EscrowError::InvalidAmount);

    // Simpan data Escrow
    ctx.accounts.populate_escrow(seed, receive, ctx.bumps.escrow)?;

    // Penyetoran token
    ctx.accounts.deposit_tokens(amount)?;

    Ok(())
}
```

Di sini kita menambahkan dua proses validasi; satu pada argumen `amount` dan satu lagi pada argumen `receive` untuk memastikan kita tidak memasukkan nilai nol untuk keduanya.

### Peringatan

Ekstensi tertentu dari SPL Token-2022, seperti transfer hooks, confidential transfers, dan default account states dapat membuat celah kerentanan seperti memblokir transfer, mengunci dana, dan menyebabkan _rug-pull_ dalam logika escrow, vault, atau CPI.

- Pastikan `mint_a` dan `mint_b` dimiliki oleh program token yang sama untuk mencegah kegagalan CPI.
- Gunakan token yang telah diaudit dengan baik (misalnya, USDC, wSOL) dari program SPL Token standar.
- Hindari mint Token-2022 yang belum diverifikasi atau kompleks.
